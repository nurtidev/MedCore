## Описание задачи

Базовый, но очень важный модуль для **защиты персональных медицинских данных**.

Медицинская платформа DCH требует строгого разграничения доступа между разными типами пользователей (врачи, координаторы, администраторы) и соответствия законодательству РК о персональных данных.

## Функциональные требования

- Жёсткое разграничение прав доступа по ролям:
  - **Врач** — доступ к своим пациентам и расписанию
  - **Координатор** — управление записями и расписанием
  - **Администратор** — управление пользователями и данными клиники
- Настройка токенов авторизации (JWT access + refresh)
- Хранение персональных данных согласно законодательству РК о персональных данных

## Ожидаемый результат

Защита чувствительной медицинской информации через надёжную систему аутентификации и авторизации с ролевой моделью.
