## Описание задачи

Внедрение возможности **оплачивать услуги прямо из системы** DCH без перехода на внешние сайты.

## Функциональные требования

- Генерация ссылок на оплату
- Проведение транзакций
- Изменение статуса услуги на "Оплачено" после успешной транзакции
- Расчёт процента от транзакций (комиссия платформы)

## Роли пользователей

| Роль | Функция |
|---|---|
| Пациент | Выполняет оплату |
| Бухгалтер | Проводит сверку платежей |
| Администратор | Управление системой |

## Интеграции с платёжными системами

- **Kaspi Pay** — основной казахстанский эквайер
- **Stripe** — международные карты

## Ожидаемый результат

Возможность принимать онлайн-оплату за медицинские услуги непосредственно из интерфейса DCH с автоматическим обновлением статусов.

## Заметки для разработки

- Нужна idempotency для всех платёжных операций (защита от дублей)
- Webhook обработка от Kaspi Pay и Stripe
- Статусная машина для транзакций: pending → processing → completed/failed
- B2B биллинг клиник (SaaS подписки) — надстройка над базовым модулем
- Генерация invoice/чеков в PDF
- Kafka события: `payment.completed`, `payment.failed`, `subscription.expired`
