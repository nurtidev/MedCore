## Описание задачи

**Подключение к внешним медицинским базам** для автоматизированного обмена данными с государственными реестрами. Получение и передача необходимой информации о пациентах без ручного ввода.

## Функциональные требования

- Обмен данными с государственными реестрами для:
  - Передачи необходимой отчётности
  - Получения статуса пациента
- Валидация ИИН через государственные базы
- Интеграция с медицинскими реестрами

## Интеграции

| Система | Назначение |
|---|---|
| **eGov API** | Государственные сервисы РК — валидация ИИН, данные граждан |
| **DAMUMED** | Медицинская информационная система РК — статусы пациентов, отчётность |

## Ожидаемый результат

Автоматизированный обмен данными с государственными реестрами для получения актуальной информации о пациентах и выполнения требований по отчётности.

## Заметки для разработки

- Circuit breaker обязателен — государственные API нестабильны
- Кэширование ответов в Redis (валидация ИИН — TTL 24h)
- Retry с exponential backoff для временных сбоев
- Все запросы к ГосAPI логировать для аудита
- ЭЦП (электронная цифровая подпись) — возможна потребность в поддержке
- Dead letter queue для неуспешных запросов
