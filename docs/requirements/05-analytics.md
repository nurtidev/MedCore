## Описание задачи

Модуль для визуализации данных с целью **обеспечить руководителей клиник прозрачной аналитикой для принятия управленческих решений**.

## Ожидаемый эффект

| Метрика | Цель |
|---|---|
| Сокращение времени администрирования | 35–50% |
| Повышение заполняемости расписания | До 85–90% |
| Выявление узких мест в воронке продаж | Да |

## Функциональные требования

### Дашборды
- Дашборды по выручке: дневные, недельные, месячные
- Аналитика расписания:
  - Заполняемость кабинетов и врачей
  - Статистика неявок (no-show rate)
- Воронки конверсии пациентов

### Экспорт и интеграции
- Экспорт графиков и данных в PDF/Excel
- Синхронизация с BI-системами через API

## Роли пользователей

| Роль | Доступ |
|---|---|
| Владелец клиники | Полная аналитика по клинике |
| Главный врач | Аналитика по врачам и расписанию |
| Суперпользователь | Аналитика по всем клиникам (SaaS) |

## Технические требования из ТЗ

- **Backend:** Node.js с агрегирующими SQL-запросами и CRON-процессами *(мы реализуем на Go + ClickHouse)*
- **Frontend:** Vue.js 2 с Chart.js или ECharts *(frontend вне скоупа MedCore)*
- Время загрузки дашбордов: **1–2 секунды**

## KPI метрики для реализации

- Загруженность врача (appointments / рабочие часы)
- Revenue per doctor
- No-show rate (% неявок)
- Заполняемость расписания (%)
- Конверсия: первичный визит → повторный
- Выручка по клинике / периоду / услуге

## Заметки для разработки

- ClickHouse — основное хранилище для аналитики (быстрые агрегации)
- Materialized views для pre-computed KPI метрик
- Kafka consumer — сбор событий из всех сервисов в реальном времени
- CRON задачи для пересчёта периодических отчётов
- REST API для дашборд данных (не gRPC — фронтенд потребляет напрямую)
- Партиционирование по месяцам в ClickHouse
